Problem Formulation

Assumptions


  • Environment - N orthogonal channels
  • Action - choose one (or two conseutive if rate-control enabled) channel to TX
  • State - observation of channels
  • Reward - -100 collide, 0 wait, +100 TX.
  • online learning
  • Predict rather than reactive

Predict - Learning from history


    Reactive-and-action require the agent to have full and perfect sensing, as well as compute out the action in a short time. Once the agent is not highly equipped, there is an obvious time gap between sensing and action, and it has to slow down or pause to make current sensing. Luckily, the DQN make actions based on the previous state, or a bunch of states, and then do the sensing. Moreover, DQN can even learn base on part of observation (partial DQN), or make an observation over a period rather than each time. Hence it is more practical and more reliable.
    The agent takes action based on the previous state, and gets an immediate reward (collide or not) and update the state, and these four variables together make up the input of DQN.

Approach

Deep Q Network

  • we adapt open-source DQN as starter, while focus on "translate" images and car motion into the <S_, A, R, S>tuple.
  • DQN output a scalar value, that later "translate" into car motion.

News
